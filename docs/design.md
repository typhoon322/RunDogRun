# v2.5 量化研究系统 — 设计说明书

> 面向小资金（<10万）的轻量级量化研究与策略监控系统

---

## 1. 系统目标

构建一个 **低频、可复现、可长期运行** 的量化研究辅助系统。

- 不做自动交易，只输出信号和报告
- 不做短线高频，只做中线（日级别）
- 不做复杂AI模型，保持可解释性

---

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| 数据优先 | 数据质量 > 模型复杂度 |
| 可复现 | 快照冻结，回测结果不变 |
| 可观测 | 全链路 Pipeline 日志 |
| 少交易 | 宁可错过，不乱交易 |
| 组合优先 | 不押单票 |

---

## 3. 系统架构

```
AKShare 数据
   ↓
数据快照 (data/snapshots/)
   ↓
市场过滤 (MA5>MA20)
   ↓
行业过滤 (Top5强行业 + 价<60)
   ↓
评分排名 (动量+低价+量+板块)
   ↓
组合分仓 (评分加权, 单票≤35%)
   ↓
快速回测 (本地CSV缓存)
   ↓
健康监控 (胜率/回撤/波动/趋势)
   ↓
策略门控 (TRADE/CAUTION/STOP)
   ↓
日报 + Streamlit UI
```

---

## 4. 核心模块

### 数据层
- **AKShare**: 全市场实时行情
- **CSV 缓存**: `data/raw/daily/` 本地历史
- **快照系统**: 每日冻结数据，保证可复现
- **数据校验**: 空值/排序/价格/日期检查

### 策略层
- **市场状态**: MA5 vs MA20，只做上升趋势
- **行业过滤**: 涨幅 Top5 行业 + 价格<60 + 动量>1.5%
- **评分排名**: 动量×0.4 + 低价×0.3 + 量×0.2 + 板块×0.1
- **组合分仓**: 评分加权，单票上限35%

### 监控层
- **健康评分**: 胜率(40) + 回撤(30) + 波动(30) = 0-100
- **策略漂移**: 检测胜率/回撤/波动的性能退化
- **Pipeline 日志**: 每一步可追溯

### 展示层
- **Streamlit UI**: 云端仪表盘，手机可访问
- **日报**: JSON + Markdown 自动生成
